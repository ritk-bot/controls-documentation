# Important Discovery About Black Pill Boards

Unlike Nucleo boards:

- Black Pill boards do **not** contain an onboard ST-Link debugger.
- STM32CubeIDE cannot directly flash programs through USB-C unless a debug probe is available.
- USB-C is connected directly to the STM32 USB peripheral.
- Flashing must be performed through DFU (Device Firmware Upgrade) mode.

This means the workflow differs significantly from Nucleo boards.

Instead of writing:

```c
HAL_GPIO_WritePin(GPIOC, GPIO_PIN_13, GPIO_PIN_SET);
```


This improves readability and survives code regeneration.

# Learning About DFU Mode

STM32 microcontrollers contain a factory bootloader in system memory.

This bootloader supports:

- USB DFU
- UART
- Other interfaces depending on MCU

When entered correctly:

```bash
lsusb
```

shows:

```text
STM32 Device in DFU Mode
```

with:

```text
VID: 0483
PID: DF11
```

---

# Understanding the Buttons

The board contains three buttons:

```text
BOOT0
NRST
KEY
```

### BOOT0

Controls boot source.

When asserted during reset:

```text
Boot from System Memory
```

which launches the STM32 factory bootloader.

### NRST

Hardware reset pin.

Equivalent to restarting the microcontroller.

### KEY

Board-specific button.

Not necessarily equivalent to BOOT0.

Its exact behavior depends on the board implementation.

---

# Entering DFU Mode

The sequence that worked:

1. Hold **BOOT0**
2. Press **NRST**
3. Release **NRST**
4. Release **BOOT0**

Verification:

```bash
lsusb
```

Output:

```text
STM32 Device in DFU Mode
```

# Fixing DFU Permissions on Linux

Created a udev rule:

```bash
sudo nano /etc/udev/rules.d/99-stm32dfu.rules
```

Contents:

```text
SUBSYSTEMS=="usb", ATTRS{idVendor}=="0483", ATTRS{idProduct}=="df11", MODE="0666"
```

Reload the rules:

```bash
sudo udevadm control --reload-rules
sudo udevadm trigger
```

Reconnect the board afterwards.

# Key Lessons Learned

## 1. The Board Was Never Actually Dead

As long as:

```text
STM32 Device in DFU Mode
```

appears in `lsusb`, the MCU is recoverable.

---

## 2. DFU Is Extremely Useful

Even without an ST-Link it is possible to:

- Read memory
- Change option bytes
- Erase flash
- Program firmware

through USB.

---

## 3. Black Pill Workflow Differs From Nucleo

### Nucleo

```text
USB → ST-Link → MCU
```

CubeIDE can flash directly.

### Black Pill

```text
USB → STM32 DFU Bootloader
```

Requires:

```text
STM32CubeProgrammer
```

unless an external ST-Link is purchased.
